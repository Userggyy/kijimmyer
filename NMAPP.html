Nmap 基本介紹

Nmap (Network Mapper) 是一款開源的網路探測和安全審計工具，用於網路探索和安全掃描。它可以發現網路上的主機和服務，並提供詳細的網路映射。

Nmap 是資安專業人員和網路管理員常用的工具，但請注意僅在合法授權的情況下使用。

主要特點

  • 主機發現 - 識別網路上的主機
  • 端口掃描 - 檢測開放端口和服務
  • 版本檢測 - 確定服務和應用程序版本
  • OS 偵測 - 識別操作系統類型
  • 腳本掃描 - 使用 NSE (Nmap Scripting Engine) 進行進階檢測

安裝方法

在 Kali Linux 中，Nmap 通常已經預裝。如果沒有，可以使用以下指令安裝：

sudo apt update
sudo apt install nmap

若要確認安裝是否成功，可以執行：

nmap --version

基本指令教學

以下是 Nmap 最常用的指令和選項：

基本掃描

nmap scanme.nmap.org

掃描特定端口

nmap -p 80,443,22 scanme.nmap.org

快速掃描

nmap -F scanme.nmap.org

服務版本檢測

nmap -sV scanme.nmap.org

小測驗

哪個選項可以用來檢測服務版本？

-sV
-O
-A
-p

解析： -sV 選項用於檢測服務版本，這對於識別潛在漏洞非常重要。

實戰範例

範例1: 掃描區域網路存活主機

nmap -sn 192.168.1.0/24

這個指令會掃描192.168.1.0/24網段中的所有存活主機。

範例2: 綜合掃描

nmap -A -T4 scanme.nmap.org

-A 啟用OS偵測、版本偵測、腳本掃描和路由追蹤，-T4 設定較快的掃描速度。

企業網路安全評估流程

  1. 使用 nmap -sn 找出所有存活主機
  2. 對關鍵伺服器執行 nmap -sV -p- 全面端口掃描
  3. 使用 nmap -A 進行深度分析
  4. 針對特定服務執行 NSE 腳本掃描

注意：這些範例僅供學習使用。在未經授權的情況下掃描網路是違法的。

小測驗

哪個指令適合快速找出網路中的存活主機？

nmap -sS
nmap -sn
nmap -sV
nmap -O

解析： -sn 選項用於主機發現，只會檢測哪些主機在線，而不進行端口掃描。

進階使用技巧

⚠️ 以下進階技巧僅適用於合法授權的滲透測試，使用前務必取得書面許可

報表輸出

繞過防火牆技巧

這些技巧可以幫助繞過常見的防火牆和IDS檢測：

# 分片掃描 + 隨機延遲
nmap -f --scan-delay 500ms target.com

# 使用誘餌IP
nmap -D RND:10 target.com

# 隱蔽掃描 (需要root權限)
sudo nmap -sS -T2 target.com

小測驗

哪個選項可以用來進行隱蔽的SYN掃描？

-sS
-sT
-sU
-sV

解析： -sS 是SYN掃描，也稱為半開放掃描，因為它不完成TCP連接，因此比完全連接掃描(-sT)更隱蔽。

NSE 腳本使用

Nmap Scripting Engine (NSE) 提供了強大的擴展功能：

# 執行特定類別的腳本
nmap --script vuln target.com

# 執行單個腳本
nmap --script http-sql-injection target.com

# 更新腳本資料庫
nmap --script-updatedb

常用腳本類別：

  • vuln: 漏洞檢測
  • exploit: 漏洞利用
  • auth: 身份驗證
  • discovery: 網路發現

報表輸出格式

Nmap 支持多種輸出格式：

# 標準輸出
nmap -oN scan.txt target.com

# XML 格式
nmap -oX scan.xml target.com

# 所有格式
nmap -oA scan target.com

輸出格式說明：

  • -oN: 普通文本格式
  • -oX: XML格式，適合機器處理
  • -oG: Grepable格式
  • -oA: 輸出所有格式

防禦方法

了解掃描技術的同時，我們也需要知道如何防禦未經授權的掃描：

1. 防火牆配置

正確配置防火牆規則，限制不必要的端口訪問：

# 示例: 使用iptables限制Nmap掃描
sudo iptables -A INPUT -p tcp --tcp-flags SYN,ACK SYN,ACK -m conntrack --ctstate NEW -j DROP
sudo iptables -A INPUT -p tcp --tcp-flags ALL NONE -j DROP

2. 端口敲擊 (Port Knocking)

隱藏服務端口，只有按特定順序訪問特定端口才會開放服務。

3. 入侵檢測系統 (IDS)

部署如Snort等IDS系統，檢測並阻止掃描行為。

4. 服務偽裝

修改服務banner，不暴露真實版本信息。

防禦網路掃描需要多層次的安全措施，包括網路層和主機層的防護。

小測驗

哪種方法可以有效隱藏服務端口，直到收到特定訪問順序？

防火牆
端口敲擊
VPN
代理伺服器

解析： 端口敲擊(Port Knocking)是一種安全技術，通過按特定順序訪問一系列關閉的端口來觸發服務端口的開放。

法律與道德規範

重要：未經授權的網路掃描是違法行為，可能導致嚴重的法律後果。

合法使用原則

  • 僅掃描自己擁有或獲得明確授權的網路
  • 獲取書面授權後再進行掃描
  • 避免在生產環境進行高強度掃描
  • 遵守所有相關法律法規

道德黑客準則

  1. 取得明確授權
  2. 不造成服務中斷
  3. 尊重隱私
  4. 負責任地披露發現的問題

本教學僅供教育目的，使用這些技術前請確保您了解並遵守當地法律。
